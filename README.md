📋 Краткое описание

Сервис читает сообщения из топиков Kafka, валидирует их и сохраняет в SQL Server. Поддерживает разные типы сущностей (продукты, материалы) с возможностью лёгкого расширения.



🚀 Запуск

bash

cd KafkaSqlBridge.Service

dotnet run

При первом запуске прочитаются все сообщения с начала.

Offset'ы автоматически сохраняются — после перезапуска чтение продолжится с места остановки.



🧩 Как добавить новый топик

Создать модель в Models/

Создать хендлер в Handlers/, реализующий IMessageHandler

Добавить метод в IDatabaseService и реализовать его

Зарегистрировать хендлер в Program.cs

Добавить топик в KafkaSettings.Topics

📌 Важно

✅ Offset'ы сохраняются через Commit() после успешной обработки

✅ Невалидные сообщения пропускаются (логируются)

✅ Поддерживается любое количество топиков
